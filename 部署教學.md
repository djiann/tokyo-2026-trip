# 東京行程指南｜打包轉移與 GitHub Pages 部署教學

把整個 `Travel` 資料夾打包帶到**個人電腦**，依本文件操作即可取得 HTTPS 連結，同伴用 Line 點開會在 Safari 正常顯示（含折疊區塊）。

---

## 一、打包帶走哪些檔案？

### ✅ 必帶

```
Travel/
├── docs/                    ← 已建好的網站（可帶；到新電腦也可重建）
├── assets/                  ← 地圖 PNG（必帶）
├── build-site.py            ← 一鍵重建網站
├── build-mobile-guide.py    ← 產生手機單檔版
├── deploy-github-pages.sh   ← 部署輔助腳本
├── .gitignore
├── tokyo-2026-guide-light.html
├── tokyo-2026-guide.html
├── journey-maps.html
└── 部署教學.md              ← 本文件
```

### ⛔ 可不帶（體積大、可重建）

| 檔案/資料夾 | 原因 |
|-------------|------|
| `Generated_image_*.png`（根目錄大圖） | 原始 AI 圖，已匯入 `assets/` |
| `.mobile-build/` | 建置暫存 |
| `.cursor/` | 編輯器設定 |
| `.DS_Store` | macOS 系統檔 |
| `tokyo-2026-guide-mobile.html` | 執行 `build-mobile-guide.py` 可重建 |
| `journey-map-data.js` / `.json` | 舊版 SVG 用地圖資料，現已不用 |

### 打包方式建議

- **zip**：右鍵 `Travel` 資料夾 → 壓縮
- 或 **AirDrop / 隨身碟 / 個人雲端**（Google Drive 個人帳號、iCloud 等）

## 二、個人電腦事前準備

1. 安裝 **Git**：https://git-scm.com/downloads  
2. 瀏覽器登入**個人** GitHub（建議無痕或個人瀏覽器設定檔，避免混到公司帳號）  
3. 確認已安裝 **Python 3**（macOS 通常內建；Windows 從 python.org 安裝）

終端機檢查：

```bash
git --version
python3 --version
```

---

## 三、在本機重建網站（建議做一次）

解壓縮後進入資料夾：

```bash
cd Travel    # 或你的實際路徑

python3 build-site.py
```

成功會看到：

```
Site ready: .../docs/
  index.html  ← 分享這個網址（同伴主入口）
  maps.html   ← 插畫地圖分頁
  guide.html  ← 完整指南（清新版）
```

### 本機預覽（選用）

```bash
cd docs
python3 -m http.server 8080
```

瀏覽器開 http://localhost:8080/ ，手機同一 Wi‑Fi 可開 `http://電腦IP:8080/`

## 四、建立個人 GitHub 倉庫

1. 登入 **個人** https://github.com  
2. 右上角 **+** → **New repository**  
3. 設定：
   - Repository name：`tokyo-2026-trip`（可自訂）
   - **Public**
   - 不要勾選 Add a README
4. 按 **Create repository**

記下你的帳號，之後網址會是：

```
https://你的帳號.github.io/tokyo-2026-trip/
```

## 五、用個人身份初始化 Git（勿改全域設定）

在 `Travel` 資料夾內：

```bash
cd Travel

git init
git branch -M main

# ⚠️ 只設定這個專案，不要加 --global
git config user.name "你的名字"
git config user.email "你的個人信箱@example.com"

git add docs/ build-site.py build-mobile-guide.py deploy-github-pages.sh .gitignore
git add tokyo-2026-guide-light.html tokyo-2026-guide.html journey-maps.html assets/
git add 部署教學.md

git commit -m "Deploy Tokyo trip guide to GitHub Pages"
```

## 六、推送到個人 GitHub

### 方式 A：HTTPS + Personal Access Token（推薦）

1. GitHub → **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**  
2. **Generate new token (classic)**  
3. 勾選 **`repo`** → Generate → **複製 token**（只顯示一次）

```bash
git remote add origin https://github.com/你的帳號/tokyo-2026-trip.git
git push -u origin main
```

提示輸入時：

- Username：你的 GitHub 帳號  
- Password：**貼 token**（不是 GitHub 登入密碼）

> 若 push 失敗且帶到公司 SSO，請確認用的是**個人帳號** token；或刪除 Keychain 裡舊的 `github.com` 憑證後重試。

### 方式 B：SSH（進階）

```bash
ssh-keygen -t ed25519 -C "你的個人信箱" -f ~/.ssh/id_ed25519_personal
```

把 `~/.ssh/id_ed25519_personal.pub` 內容貼到 GitHub → **Settings → SSH and GPG keys → New SSH key**

`~/.ssh/config` 新增：

```
Host github.com-personal
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_personal
```

```bash
git remote add origin git@github.com-personal:你的帳號/tokyo-2026-trip.git
git push -u origin main
```

## 七、開啟 GitHub Pages

1. 打開 repo 頁面  
2. **Settings** → 左側 **Pages**  
3. **Build and deployment**：
   - Source：**Deploy from a branch**
   - Branch：**main**
   - Folder：**/docs**
4. **Save**

等 **1～2 分鐘**，重新整理 Pages 頁面，會出現：

```
https://你的帳號.github.io/tokyo-2026-trip/
```

### 分享給同伴

| 連結 | 用途 |
|------|------|
| `https://你的帳號.github.io/tokyo-2026-trip/` | **主入口**（手機版，折疊+地圖） |
| `.../maps.html` | 插畫地圖分頁 |
| `.../guide.html` | 完整清新版指南 |

Line 傳主連結即可；點開會用 Safari / 預設瀏覽器開啟，折疊可正常點擊。  
建議同伴：**分享 → 加入主畫面**，出發後也能快開。

## 八、之後修改行程

1. 在本機編輯 `tokyo-2026-guide-light.html`（或 `tokyo-2026-guide.html`）  
2. 重建並推送：

```bash
cd Travel
python3 build-site.py

git add docs/
git commit -m "Update itinerary"
git push
```

約 1 分鐘後同伴重新整理網頁即為新版。

若也要更新手機單檔 HTML（非網頁版）：

```bash
python3 build-mobile-guide.py
# 產生 tokyo-2026-guide-mobile.html，可另傳 Line
```

## 九、沒有 Git 或公司網路擋 push？

### 網頁上傳（Plan B）

1. 個人 GitHub 建好空 repo  
2. **Add file → Upload files**  
3. 把 `docs/` 裡**所有檔案**拖進去（含 `assets` 子資料夾）  
4. Commit  
5. Settings → Pages → Branch: **main**，Folder: **/ (root)**（因為檔案在 root 而非 docs 子目錄）

之後每次更新需重新上傳；有 Git 會方便很多。

## 十、常見問題

### Q：同伴折疊還是打不開？

- 確認傳的是 **https 網址**，不是本機 `.html` 檔  
- 請用 Safari 開，並「加入主畫面」

### Q：圖片破圖？

- 執行 `python3 build-site.py` 確認 `docs/assets/` 有 8 張 PNG  
- push 後確認 GitHub 上 `docs/assets/` 存在

### Q：Pages 顯示 404？

- 確認 Pages 設定為 **main** + **/docs**  
- `docs/index.html` 必須存在  
- 等 2～5 分鐘再試

### Q：想設成私密，只給家人？

- 個人免費帳號：**Private repo 也可開 Pages**（2020 年起支援）  
- 建 repo 時選 **Private**，Pages 設定相同；網址知道的人仍可開啟（無 GitHub 登入牆），若需密碼保護要用 Netlify 等進階方案

### Q：公司電腦 vs 個人電腦？

- 本教學假設在**個人電腦**操作，用**個人 GitHub**  
- 勿把 repo 建在公司 organization 下  
- 此專案 `git config` 請只用**本資料夾**設定，不要用 `--global` 覆蓋公司設定
